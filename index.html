What is OpenVivoe?

VIVOE is the name of the Defense Standard 00-82 military norm, written by the UK's Ministry of Defence, a military standard to multicast video streams between device inside a terrestrial vehicles over a ethernet network (Vetronics Infrastructure for Video Over Ethernet). OpenVivoe is its open source implementation realized by OpenWide Ingenierie, a French Open Source architecte.

Multicast video streams

VIVOE aims to multicast video streams between devices over the same network, inside the same vehicle. It means that one video source can send it stream to many video sinks using a unique mulicast IP address. OpenVivoe uses Real Time Streaming Protocol (RTP) to carry those videos. To automatically play the streams, OpenVivoe implements its own version of the SAP/SDP protocol.

Manage the streams with SNMP

Not only OpenVivoe implement steaming, but it uses Simple Network Management Protocole to control it. Indeed, with OpenVivoe, one of the device remaining in the network is defined as the "Manager". From the manager you can remotely start and stop the streams on other devices, choose wich screens will displays which streams.

Linux Based

OpenVivoe has been developped on Linux. For now, only Linux version is available. However, OpenViveo could be multi-platform in a close the future.

Still under development

Please keep in mind that OpenVivoe is still under development, no stable version has been released yet. In the current version, Region Of Interest is not implemented.

Easy to customise

OpenVivoe is a free, open source software. You can download it, fork it, try it, modify it as you want!

Dependencies

  • Requires Cmake 3.5.0
  • Streaming based on Gstreamer 1.6.3
  • Configuration based on Glib 2.46.2
  • Management based on NET-SNMP 5.7.3

Get Started

Debian Based Distribution

To install and use OpenVivoe on Debian based distribution follow this steps: 


   ~$ sudo apt-get install gstreamer gst-libav gst-plugins-base gst-plugins-base-libs gst-plugins-good gst-plugins-ugly
   ~$ sudo apt-get install glib2
   ~$ sudo apt-get install net-snmp

Once depedencies are installed, clone the master branch of the GitHub repository. Or simply click on the Download button

    
	~$ git clone https://github.com/Openwide-Ingenierie/openvivoe.git

Then, just build the project


	~$ cmake -G "Unix Makefiles"
	~$ make

If you prefer, you can build a Release version


	~$ cmake -DCMAKE_BUILD_TYPE=Release -G "Unix Makefiles"
	~$ make

Try OpenVivoe

Configure net-snmp

First, you need to configure net-snmp for OpenVivoe. Net-SNMP is the library used by VIVOE to implement SNMP, the protocol used to managed the streams. SNMPv2 protcole works with a Management Information Base "MIB", which acts as a data base. VIVOE implements its own. Furthermore, Net-SNMP cames with several configuration files. To simplify the configuration OpenVivoe already wrote all those files. First, create a directory ".snmp" in your home folder and copy the content of the conf directory in there.


	~$ make $HOME/.snmp
	~$ mv conf/* $HOME/.snmp

Congratulations, your openvivoe is fully configured. However, you can take a look to configuration files, especially to vivoe-mib.conf, that you can use to fill your some of your MIB's parameters.

Launch OpenVivoe and Start Streaming

Now you can try OpenVivoe. To do so, you first launch the biary file. This binary is located in openvivoe/bin/Debug or in openvivoe/bin/Release depending the OpenVivoe's version you built. In the following example debug's version is used, as it is the default selected version when building. First, connect a webcam to your computer. In this example we assume that you only have one device available for testing. To test openvivoe you will stream from your computer to ... your computer. Now, you can start the program.


	$~ cd bin/Debug && ./openvivoe

The program should be started, and the streams should have started. If your terminal look to something like this, then openvivoe has been successfully started! Congratulations!


	~$ ./openvivoe
	NET-SNMP version 5.7.3 AgentX subagent connected
	openvivoe is up and running
	Now playing
